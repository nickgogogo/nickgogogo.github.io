HashCode理解

Posted at 2017-10-30

0：HashCode是什么

HashCode是根据对象产生的int型特征值，主要用途是通过使用HashCode查找大大降低匹配速率；

具体产生算法不固定（Oracle鼓励使用者开发自己的HashCode算法）,在这里我们主要讨论JDK 8的默认算法

1：HashCode原理

很多人误以为HashCode是对对象内存地址进行计算 X错误

在查看过source codes后发现，对于不同类型的数据对象，HashCode的产生方式是不同的。

JDK封装了Hashcode方法。经数学计算，对象会产生一个int型HashCode

这里列举两种类型数据的HashCode计算方式

1. The hash code for a String object is computed as

public int hashCode() {
int h = hash;
if (h == 0 && value.length > 0) {
char val[] = value;
for (int i = 0; i < value.length; i++) {
h = 31 h + val[i];
}
hash = h;
}
return h;
}
数学表达式为：
> s[0] 31^(n-1) + s[1]31^(n-2) + … + s[n-1]


n为String对象字符长度，h是当前字符的ASCII code


2. Integer型对象的HashCode返回值是其本身



*综上，默认算法中HashCode和内存地址没半毛钱关系！

2：HashCode冲突(collisions)

根据HashCode的生成算法，我们发现HashCode并不是唯一的。例如字符串”Aa“与”BB“产生相同的HashCode，因此永远不要把HashCode作为主键

3：用途与特性

下面这张图帮助对hashcode有一个感性的认识


HashCode与对象并不是唯一对应的：

1.相等的对象必然拥有相同的HashCode

2.不相等的对象可能拥有相同的HaschCode

3.HashCode相同的对象不一定相等

基于这些特性，在对象A,B匹配时，先匹配两个对象的HashCode，若相同，再进行进一步匹配；若不同，则两对象不同。从而大大节省了比较次数，提高查找效率
