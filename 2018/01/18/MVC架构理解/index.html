MVC架构理解

Posted at 2018-01-18

上学期在Ruby on rails开发中接触到了MVC结构的概念。因为第一次听说，而且概念也比较抽象，不是很了解其中的含义。

今天再次审视MVC框架，发现应用如此之广。

在不同的系统中，M-V-C的含义也是截然不同的

Model-View-Controller (MVC)概念第一次于1970年代后期，在伊利诺伊香槟大学的一篇论文中被提及的[Burbeck92]。

先给出定义:

Model
The model manages the behavior and data of the application domain, responds to requests for information about its state (usually from the view), and responds to instructions to change state (usually from the controller).

View The view manages the display of information.

Controller The controller interprets the mouse and keyboard inputs from the user, informing the model and/or the view to change as appropriate.

  • View比较好理解，在软件项目中，一般可理解为GUI。而相对抽象的是Model与Controller。

  • Model是目的，是整个系统的关键，也是系统存在的意义。在Model层程序调用方法、实现功能并解决问题。

这里引用Steven C. McConnell 的话，Models represent knowledge. A model could be a single object (rather uninteresting), or it could be some structure of objects.

  • Controller是User与系统之间的连接，通过控制View呈现自己，并给用户输入手段使用系统。是介于View与Model的桥梁、纽带。
